# Rentiqo Information Architecture (MVP)

## 1. Navigation model

Primary mobile navigation (bottom tabs):
1. **Discover** (map/list search)
2. **Saved** (saved homes and saved searches)
3. **Inbox** (contacts, tour requests, notifications)
4. **Profile** (account, preferences, calculators)

Contextual overlays:
- Filter sheet
- Listing media gallery
- Contact agent modal
- Tour request flow

## 2. Core content hierarchy

## Discover
- Search header (location + quick filters)
- Map/list toggle
- Active filter chips
- Result cards

## Listing detail
- Media carousel
- Price and status block
- Key property facts
- Description and highlights
- History and neighborhood snippets
- Action bar (save, contact, request tour)

## Saved
- Saved homes list
- Saved searches list
- Alert management

## Inbox
- Lead and message summaries
- Tour request status
- System notifications (alerts and updates)

## Profile
- Account settings
- Notification settings
- Search preferences
- Affordability calculator

## 3. Agent-facing IA (MVP baseline)

Agent app/web sections:
1. Lead queue
2. Lead detail
3. Profile and verification
4. Performance summary (basic)

## 4. Admin IA (internal console baseline)

1. Moderation queue
2. Listing quality dashboard
3. User support case view
4. Audit trail viewer

## 5. IA principles

- Prioritize search-to-action speed.
- Keep high-intent actions persistent and visible.
- Preserve context when moving between map, list, and detail.
- Design for progressive disclosure: summarize first, detail on demand.
