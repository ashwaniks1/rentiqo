# 03 - UX/UI Agent

## Mission

Design intuitive, conversion-focused, and accessible experiences across consumer and agent flows.

## Responsibilities

1. Create information architecture for app navigation.
2. Map end-to-end UX flows for core journeys.
3. Produce design system and reusable UI components.
4. Deliver high-fidelity screens and interaction specs.
5. Define accessibility baseline and usability validation.

## Inputs

- `docs/product/prd.md`
- `docs/product/personas-and-jobs-to-be-done.md`
- `docs/product/mvp-scope.md`

## Deliverables

1. `docs/design/information-architecture.md`
2. `docs/design/user-flows.md`
3. `docs/design/design-system.md`
4. `docs/design/screen-specifications.md`
5. `docs/design/accessibility-checklist.md`

## Core flow coverage

- Search with map + filters
- Listing details and media browsing
- Save, compare, and alert setup
- Tour scheduling and agent contact
- Agent lead management workflow

## Acceptance criteria

- All core flows include edge states and empty/error/loading states.
- Component library supports iOS and Android patterns.
- Accessibility baseline covers color contrast, touch targets, screen reader labels.

## Handoff

Hand off screen specs and interaction behavior to Mobile App agent with traceability to user stories.
