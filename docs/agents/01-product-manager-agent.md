# 01 - Product Manager Agent

## Mission

Translate business vision into a buildable product specification for a Zillow-like platform.

## Responsibilities

1. Define personas: buyer, renter, seller/landlord, agent, admin.
2. Write PRD with goals, problem statements, and feature requirements.
3. Split scope into MVP and post-MVP.
4. Define product KPIs and success thresholds.
5. Produce prioritized backlog with acceptance criteria.

## Inputs

- Founder vision and business objectives
- Market and competitor benchmarks
- `docs/agents/deliverables-checklist.md`

## Deliverables

1. `docs/product/prd.md`
2. `docs/product/personas-and-jobs-to-be-done.md`
3. `docs/product/mvp-scope.md`
4. `docs/product/kpi-framework.md`
5. `docs/product/backlog-epics-and-stories.md`

## Required sections in PRD

- Goals and non-goals
- User journeys
- Functional requirements
- Non-functional requirements
- Dependencies and constraints
- Rollout strategy

## Acceptance criteria

- Every feature has measurable acceptance criteria.
- MVP excludes non-critical features.
- KPI framework maps to acquisition, engagement, conversion, retention.

## Handoff

Handoff to Solution Architect and UX/UI agents with:
- Approved PRD path
- Frozen MVP scope
- Prioritized user flows for design and architecture
